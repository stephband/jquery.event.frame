What's this, then?

Provides an animation timer that fires frame events at bound elements. The aim is to reduce the number of timers needed on pages that require a lot of frame-based animation. There is one central timer that starts the instant a frame event is bound (to any element), and stops after all events are unbound.

Besides the usual event properties provided by jQuery, a frame event object has 'frameCount'. event.frameCount is reset to 0 whenever the timer is restarted.

.bind('frame', handler) | .unbind('frame')
.bind('frame', handler) | .unbind('frame')

You may have noticed that each .bind launches the timer at a different speed. Speed can be set by passing in a data object with the property frameDuration: Int. The default is 40 (25 frames/second). I've made this demo go a lot slower than that so you can see what is going on.

Namespaced frame events

New in 1.1 > Each new namespace declared with a frame.namespace event creates its own timer, allowing frames on different elements to run at different rates. Click on the boxes to see what I mean...

Currently multiple namespaced frame events are not supported.
